# v0.2.3 — Tightened basis definition + stronger tint + closing gets tinted

Date: 2026-04-22
Scope: backend writing prompts + frontend LetterPanel
Status: implemented, re-generated dr_hu_eb1a/judging end-to-end with DeepSeek

---

## Task Understanding

Follow-up on v0.2.2 feedback from the user:

1. **Base tint too pale.** `bg-blue-50` / `bg-yellow-50` were not visible
   enough as persistent background; bump one shade.
2. **Closing sentence never turned yellow.** v0.2.2 excluded
   opening/closing from basis tinting, but these framing sentences are
   by nature inference. The user expected them tinted.
3. **"Cited a snippet" ≠ "evidence statement".** v0.2.2's prompt
   encouraged the LLM to label anything with a citation as `evidence`.
   The user's refined rule: a sentence is **evidence** only when it
   reports a fact; sentences that reason about, characterize, or draw
   conclusions from evidence are **inference** even if they also cite
   the record.

## Implementation Plan

1. Rewrite the basis definition in both prompt paths to be behavior-based,
   not citation-based. Add a concrete RULE OF THUMB: strip the citation
   and see what is left — factual report → evidence, argumentative claim
   → inference. Explicit marker list ("indicates", "demonstrates",
   "establishes", "evidences", …) so the LLM recognizes the pattern.
2. Bump `LetterPanel` color classes: base moves from `-50` to `-100`,
   hover from `-100` to `-200`, focused from `-200` to `-300`, for both
   evidence (blue) and inference (yellow).
3. Remove the `isBody` gate in `LetterPanel` so that opening and closing
   sentences also receive their basis tint (hard-coded to `"inference"`
   upstream, so they render yellow).

## Changes Made

### Backend
- `backend/app/services/petition_writer_v3.py` —
  - Rewrote the `basis` instructions in `_write_single_subargument`'s
    user prompt and in `_step1_generate_argument_body`'s user prompt.
    Both now lead with "this is about what the sentence *does*, not
    whether it cites a snippet", list interpretive verbs that should
    trigger `inference`, and close with the strip-the-citation
    rule-of-thumb plus an explicit warning not to default to `evidence`.

### Frontend
- `frontend/src/components/LetterPanel.tsx` —
  - Dropped the `isBody` narrowing so every sentence computes a `basis`.
  - New tint classes:
    - evidence: `bg-blue-100` base · `hover:bg-blue-200` · focused `bg-blue-300`
    - inference: `bg-yellow-100` base · `hover:bg-yellow-200` · focused `bg-yellow-300`

## Risks / Pending

- **Browser hard refresh required** to load new bundle and re-fetch
  writing JSON; the previous dev session rendered v0.2.2's cached data
  (which was why the user saw no yellow on the closing).
- **Prompt sensitivity**: DeepSeek's labeling still depends on prompt
  compliance. The new prompt nudges toward recognizing interpretive
  verbs, but spot-checking the first few generated sections is advised
  before trusting at scale.
- **Legacy data fallback remains snippet-based**: for sections
  generated before v0.2.2 whose JSON lacks the `basis` field, the
  frontend still infers basis from `snippet_ids` presence. This is a
  coarse proxy; the only reliable fix is to regenerate.
- **Pre-existing data-dir mismatch** (out of scope here): writing
  save/load uses `petition_writer_v3.PROJECTS_DIR` which is hardcoded
  to the repo's `backend/data/projects`, while project metadata uses
  `PETITON_PROJECTS_DIR=/root/petiton-data/projects`. Happens to be
  kept in sync today but a future bug nursery. Flag for a later
  cleanup.
- **Rollback**: `git revert` this commit. Generated JSON with `basis`
  remains backward-compatible with the v0.2.2 frontend.
