# v0.1.0 — Strip userstudy shell, promote DemoPage to sole route

Date: 2026-04-22
Scope: frontend only
Status: implemented, verified via dev server module compilation

---

## Task Understanding

- **Goal**: Convert the userstudy demo frontend into a clean development baseline.
  Keep the three-column Write Mode interface (Evidence Cards + PDF / Write Tree /
  Letter Panel) embodied by `DemoPage` as the only route. Remove userstudy-specific
  scaffolding that is no longer needed.
- **Scope of change**: frontend only.
  - Replace `/mapping` + `/video` routing with a single `/` route that renders `DemoPage`.
  - Delete the userstudy page component (`VideoPage.tsx`).
  - Reconstruct build infrastructure inside `frontend/` (package.json, vite.config.ts,
    tsconfig*, index.html, eslint.config.js, public/) because the original project
    placed these at repo root and they were archived in v0.0 (the initial archive
    pack) — leaving `frontend/` without a working build.
- **What was NOT changed**:
  - `backend/` — untouched.
  - `backend/data/projects/dr_hu_eb1a/` — retained (may still be referenced later).
  - `src/components/`, `src/context/`, `src/services/`, `src/hooks/`, `src/constants/`,
    `src/types/`, `src/data/`, `src/i18n/`, `src/assets/` — all retained even if
    individual files are not transitively reachable from DemoPage. Cleaning these
    is deferred to a later task to keep this change atomic (DeveRule Rule 3/4/28).
  - `src/video/drHuVideoScenario.ts` and `src/video/videoDemoPresets.ts` — retained.
    They are imported by `EvidenceCardPool`, `ArgumentGraph`, `LetterPanel`, and
    `DocumentViewer`. In the DemoPage routing context their route-detection helpers
    return false by design, so the userstudy branches remain dormant without
    requiring component refactoring in this change.
  - `archive.tar.gz` — untouched; remains the recovery source.
- **Assumptions** (confirmed with user):
  - "phase2 实机演示环节" refers to `DemoPage` — the three-column Write Mode
    interface with mock data.
  - A single route is sufficient for now; routes and subroutes can be added later.
  - Unused component cleanup is out of scope for this version.

## Implementation Plan

1. Verify no external references would break when `VideoPage.tsx` is removed.
2. Rewrite `src/App.tsx` to route `/` to `DemoPage` and `*` to `/`, removing
   `AppProvider`, `VideoPage`, and the `/mapping` + `/video` routes.
3. Delete `src/video/VideoPage.tsx` and the root-level `VideoDemoScript (1).pdf`.
4. Reconstruct build infrastructure in `frontend/` (copy from archive.tar.gz).
5. Patch `vite.config.ts` to drop the archived `app/index.html` secondary entry.
6. Run `npm install`; start `npm run dev`; verify module compilation for the
   DemoPage component tree.
7. Write this log.

## Verification

- `npm install` → 298 packages, 0 vulnerabilities.
- `npm run dev` → Vite v7.3.2 ready on `http://localhost:5174/`, no warnings
  or errors in the startup log.
- Direct module fetch via curl returned HTTP 200 for the DemoPage module graph:
  `App.tsx`, `main.tsx`, `components/index.ts`, `demo/DemoPage.tsx`,
  `demo/mockData.ts`, `components/EvidenceCardPool.tsx`,
  `components/ArgumentGraph.tsx`, `components/LetterPanel.tsx`,
  `components/ConnectionLines.tsx`, and all five contexts
  (`ProjectContext`, `SnippetsContext`, `ArgumentsContext`, `UIContext`,
  `WritingContext`). No compilation errors emitted by Vite during these fetches.
- `npx tsc --noEmit -p tsconfig.app.json` reports **pre-existing** strict-mode
  errors (unused locals, type mismatches) across ~15 files. These errors
  originated in the untouched source tree and are not introduced by this change.
  They do not block Vite dev mode (esbuild performs type stripping only).
  Cleaning them up is a separate task.

## Risks / Pending

- **In-browser runtime**: I verified that every module in the DemoPage graph
  compiles and is served by Vite, but I did not open the page in a real browser.
  The user should visit `http://<server>:5174/` (or set up the reverse-proxy /
  port forward) and confirm the three-column layout renders with mock data.
- **Pre-existing TS strict errors**: `npm run build` will fail until these are
  fixed. Dev mode works today. Recommend opening a follow-up task
  (`v0.2.x-fix-strict-ts-errors`).
- **Dormant userstudy branches**: `isDrHuVideoRoute`, `isDrHuVideoExhibitInteractive`,
  `DR_HU_VIDEO_STANDARD_ORDER`, and related imports still exist inside the shared
  components. They evaluate to false in this routing configuration, but should
  eventually be removed when components are refactored.
- **Unused components and services**: Many files in `src/components/`, `src/hooks/`,
  `src/services/`, `src/data/` are not transitively used by DemoPage. They were
  intentionally kept to avoid scope creep. A follow-up audit should identify and
  delete dead code.
- **Rollback**: every deleted file exists inside `archive.tar.gz` at the repo root.
  Reverting this change requires restoring `App.tsx` and `src/video/VideoPage.tsx`.
